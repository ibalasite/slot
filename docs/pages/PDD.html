§1 Design Vision（設計願景）

1.1 主題定義

Thunder Blessing 以希臘神話為核心敘事框架，世界觀設定在眾神之巔奧林匹斯山。Zeus（宙斯）是最高神祇，掌管閃電與天空；Pegasus（天馬）是翼馬神獸；Athena（雅典娜）是智慧與戰略女神；Eagle（雄鷹）是宙斯的神聖信使。

遊戲符號系統映射神話層級：

  • Wild（百搭）：Divine Lightning（神聖閃電）／Golden Thunderbolt（黃金雷霆），Zeus 的神聖力量象徵物，可代替一切（除 Scatter）
  • Scatter：Thunder Blessing 符文，具有引爆閃電力量的神諭符號
  • P1（最高賠）：Zeus（宙斯），雷電之神
  • P2：Pegasus（天馬），翼馬神獸
  • P3：Athena（雅典娜），智慧與戰略女神
  • P4：Eagle（雄鷹），宙斯的神聖信使
  • Low 符號（L1–L4）：Zeus 名字的四個字母 Z / E / U / S，以金色銘文呈現

1.2 視覺風格方向

史詩電影感（Epic Cinematic）

主視覺定調為「神殿巔峰，雷霆降臨」：

  • 宏大的立體空間感，滾輪坐落於帕德嫩神殿列柱之間
  • 雲層與光影動態渲染，呈現時序從黃昏到雷暴的戲劇性氛圍
  • 閃電特效以體積光（Volumetric Light）呈現，光錐從天空貫穿雲層照射神殿
  • 金屬鍛刻質感的符號，帶有浮雕凸紋、鍍金邊框與皮革紋理細節
  • 整體色調偏向史詩繪畫：琥珀金、深海藍、皇家紫，配以白色閃電輝光

1.3 設計原則

  1. 層次即敘事：每一個視覺元素的尺寸、亮度、動態幅度都服務於「哪件事最重要」——Wild 最亮、Scatter 最動態、Low 符號最克制。
  2. 張力累積可視化：閃電標記的累積、FREE 字母的亮燈、倍率進度條的填充，都是視覺化的「能量蓄積」，驅動玩家情緒跟著遊戲進程高漲。
  3. 特效有目的，不濫用：每種特效只在對應的遊戲事件觸發時出現，絕不為裝飾而插播；動畫播放不阻礙遊戲節奏。
  4. 一致的神話美學語言：所有 UI 元素（按鈕、HUD、對話框、進度條）均使用同一套希臘神話風格語彙——石刻字體、鍛金框架、神殿柱式裝飾。
  5. 行動優先的可讀性：在 320px 至 1440px 的所有尺寸下，關鍵資訊（餘額、押注、WIN 值）必須無障礙易讀，符號圖案必須可辨識。

§4 Lightning Mark Design（閃電標記設計）

4.1 閃電標記視覺外觀

單一標記

  • 形狀：黃金閃電叉圖案（Zeus 閃電符號，比例 1:1）
  • 尺寸：覆蓋符號格子面積的 40%，疊加於符號右上角
  • 顏色：黃金色（#FFD700）帶白色電弧光暈
  • 效果：標記本身有持續的電弧粒子從頂端散落（particle system，低強度）
  • 透明度：符號底部仍可見，標記以 60% 不透明度疊加，不完全遮蔽符號

標記生成動畫

  • 從符號消除位置中心爆出（scale 0 → 1.2 → 1.0，Ease Out Back）
  • 時長：0.4s
  • 同步：與 Cascade 消除動畫同步，消除完成後 0.1s 出現標記

4.2 累積顯示（進度指示）

當盤面存在閃電標記時，滾輪框架右上角出現閃電標記計數器：

4.3 Thunder Blessing Scatter 觸發視覺序列

  1. Scatter 落盤（0.0s）：SC 符號落定，電弧從 SC 位置向外擴散
  2. 感應動畫（0.2s）：盤面所有閃電標記同步閃爍，SC 與各標記之間出現電弧連線
  3. 引爆開始（0.8s）：所有閃電標記同時爆炸（金色光球），背景閃白光
  4. 符號轉換（1.2s）：標記格符號以「碎片化 → 重組」方式替換為目標符號
  5. 完成顯示（1.8s）：新符號放大落定，整體盤面金色光暈持續 0.5s
  6. 第二擊（若觸發）（2.3s）：再次脈衝，符號依升階路徑再次轉換，播放升階動畫
  7. 重新判定（3.0s）：恢復正常狀態，重新計算連線

§5 Cascade Animation Design（連鎖消除動畫設計）

5.1 符號消除動畫

標準消除（無雷霆祝福）

  • 碎片材質：金屬碎片，帶各符號主色調，重力加速度下落
  • 消除完成：格子透明，等待新符號填入
  • 時長：消除動畫 0.5s + 碎片殘留 0.3s（後台可繼續計算）

  • 電弧爆炸效果，碎片為電弧粒子（藍白色閃電碎片）

  • 時長：1.0s（含電弧殘留效果）

5.2 新符號落下動畫

  • 落下物理：彈性碰撞感（Ease Out Bounce），模擬重力加速度
  • 每個新符號獨立時序，從上方（或滾輪頂端遮罩外）落入
  • 落下速度：約 60px/frame（60fps），格子間有 0.05s 延遲（階梯效果）
  • 落定後：輕微彈跳（scale 1.0 → 1.05 → 1.0，0.15s）
  • 新落入符號在第一幀有短暫半透明（opacity 0→1，0.1s）

5.3 滾輪行數擴展動畫

FREE 字母亮燈規則

Cascade 次數 亮燈字母 配色

第 2 次 R 金色點亮，有光暈

§14 Design Delivery Requirements（設計交付規格）

14.1 資源格式

資源類型 格式 說明
靜態符號（Idle 幀） PNG-32 帶 Alpha 透明通道，最大 512×512px
動態符號（Win / Special） Spine 骨架動畫（.json + .atlas） 使用 Spine 2D，版本 ≥ 4.1
UI 元素（按鈕、框架） PNG-32 / SVG 小圖示用 SVG，大圖用 PNG
背景圖層 PNG-32（橫屏）/ WebP（行動端） 多解析度輸出：1x / 2x / 3x
粒子特效資源 PNG Sprite Sheet 64×64px 至 128×128px 單幀
字體 WOFF2 + TTF 子集化（Subset），僅包含遊戲所需字符
音效（見 AUDIO.md） OGG + MP3 雙格式確保瀏覽器兼容性

14.2 幀率規格

  • 目標幀率：60fps（所有動畫設計以 60fps 為基準）
  • 行動裝置最低要求：30fps（降級模式，關閉部分粒子特效）
  • Spine 動畫：所有 Win / Special 動畫以 60fps 設計關鍵幀
  • 粒子系統：Max 500 粒子（桌面），Max 200 粒子（行動裝置）
  • 動畫曲線基準：Ease Out Cubic（cubic-bezier(0.33, 1, 0.68, 1)）為預設
