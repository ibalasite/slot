§1 Design Vision（設計願景）

1.1 主題定義

Thunder Blessing 以希臘神話為核心敘事框架，世界觀設定在眾神之巔奧林匹斯山。Zeus（宙斯）是最高神祇，掌管閃電與天空；Pegasus（天馬）是翼馬神獸；Athena（雅典娜）是智慧與戰略女神；Eagle（雄鷹）是宙斯的神聖信使。

遊戲符號系統映射神話層級：

  • Wild（百搭）：Divine Lightning（神聖閃電）／Golden Thunderbolt（黃金雷霆），Zeus 的神聖力量象徵物，可代替一切（除 Scatter）
  • Scatter：Thunder Blessing 符文，具有引爆閃電力量的神諭符號
  • P1（最高賠）：Zeus（宙斯），雷電之神
  • P2：Pegasus（天馬），翼馬神獸
  • P3：Athena（雅典娜），智慧與戰略女神
  • P4：Eagle（雄鷹），宙斯的神聖信使
  • Low 符號（L1–L4）：Zeus 名字的四個字母 Z / E / U / S，以金色銘文呈現

1.2 視覺風格方向

史詩電影感（Epic Cinematic）

主視覺定調為「神殿巔峰，雷霆降臨」：

  • 宏大的立體空間感，滾輪坐落於帕德嫩神殿列柱之間
  • 雲層與光影動態渲染，呈現時序從黃昏到雷暴的戲劇性氛圍
  • 閃電特效以體積光（Volumetric Light）呈現，光錐從天空貫穿雲層照射神殿
  • 金屬鍛刻質感的符號，帶有浮雕凸紋、鍍金邊框與皮革紋理細節
  • 整體色調偏向史詩繪畫：琥珀金、深海藍、皇家紫，配以白色閃電輝光

1.3 設計原則

  1. 層次即敘事：每一個視覺元素的尺寸、亮度、動態幅度都服務於「哪件事最重要」——Wild 最亮、Scatter 最動態、Low 符號最克制。
  2. 張力累積可視化：閃電標記的累積、FREE 字母的亮燈、倍率進度條的填充，都是視覺化的「能量蓄積」，驅動玩家情緒跟著遊戲進程高漲。
  3. 特效有目的，不濫用：每種特效只在對應的遊戲事件觸發時出現，絕不為裝飾而插播；動畫播放不阻礙遊戲節奏。
  4. 一致的神話美學語言：所有 UI 元素（按鈕、HUD、對話框、進度條）均使用同一套希臘神話風格語彙——石刻字體、鍛金框架、神殿柱式裝飾。
  5. 行動優先的可讀性：在 320px 至 1440px 的所有尺寸下，關鍵資訊（餘額、押注、WIN 值）必須無障礙易讀，符號圖案必須可辨識。

§4 Lightning Mark Design（閃電標記設計）

4.1 閃電標記視覺外觀

單一標記

  • 形狀：黃金閃電叉圖案（Zeus 閃電符號，比例 1:1）
  • 尺寸：覆蓋符號格子面積的 40%，疊加於符號右上角
  • 顏色：黃金色（#FFD700）帶白色電弧光暈
  • 效果：標記本身有持續的電弧粒子從頂端散落（particle system，低強度）
  • 透明度：符號底部仍可見，標記以 60% 不透明度疊加，不完全遮蔽符號

標記生成動畫

  • 從符號消除位置中心爆出（scale 0 → 1.2 → 1.0，Ease Out Back）
  • 時長：0.4s
  • 同步：與 Cascade 消除動畫同步，消除完成後 0.1s 出現標記
